# 🧊 Cartel Watch — 横並び値上げ監視ダッシュボード

「表向きは競合なのに、同時期・同程度に値上げしている企業群（＝協調的行為の候補）」を、
**業界 × 商品カテゴリ × 企業 × 値段 × 値上げ時期** の軸で可視化するツールです。
アイス6社へのカルテル疑い立ち入り検査（2026年）のような構図を、他業界でも探せるように作りました。

## 使い方

`overview.html` を**ブラウザで開くだけ**です（インストール・ビルド・通信不要）。
データは `data.js` / `cases-data.js` に分離してあるので、誰でも追記できます。

## 画面構成（4ページ）

| ページ | 内容 |
|--------|------|
| **⓪ 総合ダッシュボード** (`overview.html`) | **摘発済み（確定）と未摘発（疑い候補）を分けて俯瞰**。ここが入口 |
| ① 公表値上げの横並び (`index.html`) | 公表された値上げの同時性を検出（疑い度ランキング・タイムライン・クロス分析） |
| ② 仕入れコスト構造 (`procurement.html`) | 見えない値上げをコスト乖離で炙り出す。仕入れ品目の連動マトリクス |
| ③ 確定カルテル事件 (`cases.html`) | 公取委の確定処分=一次情報（76件、答え合わせ） |

### ⓪ 摘発済み / 未摘発 の切り分け
- **🔴 摘発済み**: `cases-data.js`（公正取引委員会の確定処分）。実際に処分されたカルテル・談合。
- **🟡 未摘発（疑い候補）**: `data.js` から①横並びクラスタ・②コスト乖離を算出。摘発されていないが疑い度が高いもの。
- 未摘発候補のうち、**その業種に過去の摘発前例がある**ものは「類似業種に摘発前例」を表示（注意度↑）。
- 予測による上振れ検出は `forecast/`（TimesFM）を参照。

## 4つの分析ビュー

| ビュー | 何が分かるか |
|--------|--------------|
| **サマリ** | 対象イベント数・検出クラスタ数・高リスク件数 |
| **同時値上げ疑いランキング** | カテゴリごとに「疑い度スコア」で並べる。列クリックで並べ替え |
| **値上げタイムライン (スイムレーン)** | 縦に揃う＝同時値上げ。原材料コスト線を重ねて"説明可能か"を確認 |
| **クロス分析** | 異なるカテゴリ（例: 小麦粉 × 冷凍食品）の値上げ時期の連動を相関係数で測る |

## 疑い度スコアの考え方

カテゴリ内で「窓（既定2ヶ月）」以内に連鎖した値上げを1つのクラスタとし、次で 0–100 点化します。

```
疑い度 = 50 × 参加率        (カテゴリ内の何社が動いたか)
       + 30 × 時間の近さ    (どれだけ短期間に集中したか)
       + 20 × 値上げ率の揃い (各社の値上げ率がどれだけ似ているか)
```

- **70以上＝高 / 40–69＝中 / 40未満＝低**
- 直前6ヶ月で関連原材料が **+10%以上** 上がっていれば「**コスト連動の可能性**」を付記
  → 共通コスト要因で説明でき、協調の疑いは相対的に下がります（＝冤罪防止）。

### クロス分析が効く理由
小麦粉のように**共通原材料に連動**するカテゴリは、横並びでも自然です。
逆に「**共通原材料が無いのに値上げ時期が強く連動**」していれば、業界横断の横並びとして要注目になります。

## ② 仕入れコスト構造ページ (`procurement.html`)

アイスのように**値上げが公表される業界**は ①で見えます。しかし**飲食・クリーニング・八百屋**などは
値上げが公表されず見えにくく、しかも談合は消費者向け価格だけでなく**仕入れ（procurement）側**でも起こり得ます。
そこで②では「見えない値上げ」と「仕入れ先談合の候補」を次の仕組みで炙り出します。

| ビュー | 何が分かるか |
|--------|--------------|
| **業態別 説明できない上乗せ** | 乖離 = 実売価格指数 − 理論コスト指数。大きいほどコストで説明できない |
| **コスト構造 vs 実売価格** | 実売（青）と仕入れバスケットから計算した理論コスト（紫）、その差（赤帯） |
| **コストバスケット内訳** | どの仕入れ品目が値上げを駆動しているか（構成比 × 上昇率） |
| **仕入れ品目 連動マトリクス** | 品目どうしの価格相関。無関係なはずが強連動＝川上（仕入れ先）談合の候補 |
| **クロス品目検証** | 小麦粉 × 野菜 などを重ね、連動度と「両方を仕入れる業態」を確認 |

### 「見えない値上げ」を測る式
```
理論コスト指数(t) = Σ（仕入れ品目の価格指数(t) × 構成比）
乖離            = 実売価格指数(t) − 理論コスト指数(t)   ← コストで説明できない上乗せ
```
- 乖離 +20pt以上＝高 / +8〜+20＝中 / それ未満＝低
- 例: 八百屋は野菜に連動して説明可（乖離小）、クリーニングは光熱費・人件費だけでは説明できない乖離が出る、など

### 仕入れデータの追記方法
```js
// data.js の procurement 内
inputs:     [{ id:"egg", name:"卵", series:[/* months と同じ長さの指数 */] }]
categories: [{ id:"cafe", name:"カフェ",
              basket:[{input:"egg",w:0.1},{input:"labor",w:0.4}, ...],  // w の合計=1.0
              actual:[/* 実売価格指数 months と同じ長さ */] }]
```
`procurement.months` の配列長と、各 `series` / `actual` の長さは必ず揃えてください。

## ③ 確定カルテル事件ページ (`cases.html`) — 一次情報

①②が「疑いの候補」を探すのに対し、③は **実際に摘発された事件**＝答え合わせの一次情報です。
公正取引委員会『独占禁止法の法的措置一覧』（令和元〜7年度）から **76件** を自動抽出して収録しています。

- 種別（**価格カルテル / 入札談合 / 不公正な取引方法 / 私的独占**）・業種・年・キーワードで絞り込み
- 摘発タイムライン、関与企業のチップ表示、公取委原文へのリンク
- 例: ごま油2社の価格引上げ合意(2025-05)、損保大手の保険料調整(2024-10)、電力大手のカルテル(2023-03)、LPガスバルブ(2024-06) など

### 一次情報の取得・更新
```bash
bash sources/fetch.sh      # 公取委サイトから再取得 → cases-data.js を更新
```
- 抽出ロジック: `sources/parse_jftc.js`（法条「3条後段」=不当な取引制限 でカルテル/談合を判定）
- 取得した原本: `sources/raw/jftc_r1.html`〜`jftc_r7.html`
- **収録できる一次情報の全リスト（CPI・企業物価・小麦・エネルギー・TDB 等）は [`sources/SOURCES.md`](sources/SOURCES.md)** を参照
- 価格系列は e-Stat API / 日銀時系列CSV から取得して `data.js` に流し込めます（手順は SOURCES.md）

## データの追記方法 (`data.js`)

```js
// 1) 企業を companies に追加（無ければ）
{ id: "newco", name: "新会社" },

// 2) 値上げイベントを events に追加
{ companyId:"newco", categoryId:"ice", product:"商品名",
  date:"2025-09", changePct:8, source:"出典(要確認)" },
```

- `date` は `"YYYY-MM"` 形式
- 新しい業界/カテゴリ/原材料は `industries` / `categories` / `commodities` に追加
- `categories[].commodities` に関連原材料IDを入れると「コスト連動」判定の対象になります

## ⚠️ 重要な注意

- 収録データは公開報道・各社発表を元にした**おおよその参考値（サンプル）**です。
  実際の検証には**一次情報の確認が必須**です。
- **横並び値上げ＝違法の証拠ではありません。** 原材料高騰など正当な共通要因がほとんどです。
- 本ツールは「どの組み合わせを詳しく調べるべきか」という**調査の出発点**を示すためのものです。
