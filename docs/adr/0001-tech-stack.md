# ADR-0001: 外販化に向けた技術スタック方針（T051）

- 日付: 2026-07-02
- ステータス: 採択

## コンテキスト

現在は静的 HTML/JS + Vercel（ビルドなし）。外販には認証・課金・アラートが必要になる。
「今すぐ全面リライトするか、段階移行するか」の意思決定。

## 決定

**段階移行方式を採る。ヒアリング（T003/T004）完了までは静的構成を維持する。**

| フェーズ | 構成 | 理由 |
|---------|------|------|
| 現在〜T004 完了 | 静的 HTML/JS + Vercel（現状維持） | 課金判断前の開発投資を最小化。デモ・LP・レポートは静的で全部作れる |
| Phase 2（課金 Go 判断後） | Next.js (App Router) + Vercel + Clerk(認証) + Stripe(課金) + Neon/Postgres(データ) | 既存 Vercel 資産・ドメインを継続。静的ページは `public/` にそのまま移設できる |
| Phase 3 | + Cron(データ更新) + Resend(メール) + API Routes(REST API) | すべて Vercel エコシステム内で完結、運用者1人でも回る |

## 却下した代替案

- **Rails / Django 等のフルスタック移行**: 運用者1人・フロント中心の製品には過剰。ホスティングも増える
- **Supabase 一体型**: 有力な次点。認証+DB が一体で安いが、Stripe 連携と RLS 設計の学習コストで Clerk+Neon に劣後（Phase 2 開始時に再評価可）
- **今すぐ Next.js 化**: 課金判断前の投資になる。ヒアリングでピボットの可能性がある間は据え置き

## 影響

- Phase 2 開始時のマイグレーション作業: 静的ページの `public/` 移設 + データの DB 化（T026）で見積もり 2〜4週間
- それまでの機能追加（レポート・LP・説明ページ）はすべて静的で実装し、移行時に資産として持ち越す
